注解复习

发表于 2022-05-07 更新于 2022-06-03 分类于 编程学习 ， Spring框架 阅读次数：

通过 hm-dianping 项目复习一下开发过程中常用的一些注解。

基础

⭐ 自动装配

什么是自动装配？使用 SpringBoot 时，会自动将 Bean 装配到容器中（依赖注入）。
友情链接：SpringBoot 自动装配原理及分析

@SpringBootApplication

@SpringBootApplication注解包括以下三个注解：

  • @ComponentScan：扫描本包下的所有路径，读取@Service等注解标识的类到 Spring 容器。
  • @Configuration：在 IoC 中讲过用法，可以通过@Bean注解往 Spring 容器中配置类。
  • @EnableAutoConfiguration：自动读取通过@Bean注解配置好的类，并且到classpath指定的路径读取配资好的类，将控制权教给 Spring 容器

@Bean 和 @Component

@Bean 注解作用于方法，产生一个 Bean 对象，交给 Spring 管理。
@Componet 注解作用于类

@Bean 主要用在@Configuration 和@Commponent 注解的类里。

@Autowired

默认按类型自动装配，相同类型有多个 bean（一个接口有多个实现类）会报错


加上@Qualifier 可以根据 byName 方式自动装配。

@Resource

默认按名称自动装配，比@Autowired 好用！不需要额外操作。

⭐友情链接：对@Resource 和@Service 使用的理解和总结

流程相关

Config

配置类中用到的注解

@Configuration

用于定义配置类，可替换 XML 配置文件。

Controller

@RestController

相当于@Controller+@ResponseBody
@Controller：单独使用，一般用于返回视图情况，属于比较传统的 Spring MVC 应用，前后端不分离。
@ResponseBody：方法返回值以指定格式写入 HTTP response body，而不是解析为跳转路径。

@PostMapping

映射 HTTP POST 请求，一般是对服务器做改变，常用来数据的提交、新增

@GetMapping

映射 HTTP GET 请求，获取数据，只查询

@PutMapping

映射 HTTP PUT 请求，和 POST 请求一样都会改变服务器数据，但 PUT 侧重对数据的修改

@DeleteMapping

映射 HTTP DELETE 请求，删除服务器资源

@RequestMapping

映射请求，可直接替代上面几个，但不便于他人对代码的阅读和理解，建议区分开来，养成好的代码习惯。

Entity/DTO

以下三个是 lombok 的注解

@Data

用在实体 bean 上，不需要写 set 和 get
= @Getter @Setter @RequiredArgsConstructor @ToString @EqualsAndHashCode

@NoArgsConstructor

无参构造函数

@AllArgsConstructor

全参构造函数

@EqualAndHashCode

引申问题：重写 equals 为什么要重写 hashcode？
生成 equals()和 hashCode()
@EqualsAndHashCode(callSuper = false)：不会比较其继承的父类的属性（只对比子类附加的属性）可能会导致错误判断；
@EqualsAndHashCode(callSuper = true)：会比较其继承的父类的属性；

@Accessors

@Accessors(chain=true)：开启链式访问

@TableName

mybatis-plus 中的注解，主要是实现实体类型和数据库中的表实现映射

Impl

@Service

标记当前类是一个 service 类，自动注入到 spring 容器

术语

  • Aspect（切面）
  • Joinpoint（连接点）
  • Pointcut（切入点）
  • Advice（通知/增强）
术语 注解 执行时间点
前置通知 @Before 在方法之前执行
后置通知 @After 在方法之后执行
异常通知 @AfterThrowing 方法出现异常时执行
最终通知 @AfterReturning 方法返回后执行
环绕通知 @Around 方法之前和之后执行
