⭐集合类架构

概述：Java集合框架由Collection（集合）和Map（图）两大类组成。前者储存元素集合，后者储存键/值对映射。

Collection接口含三个子类：List、Set和Queue。具体实现类有ArrayList、LinkedList、HashSet、HashMap等等。

注意：Map继承自Object，与上三者不同

集合框架是一个带白哦和操纵集合的统一架构，其包含如下内容：

  • 接口：集合的抽象数据类型（例如Collection、List、Set、Map…）
  • 实现：集合接口的具体实现（例如ArrayList、LinkedList…）
  • 算法：实现集合接口的对象里方法执行的一些计算（例如搜索和排序）

除了集合，该框架也包含几个Map接口和类。虽存储的是键/值对，但也整合在集合中。

友情链接：Java 集合框架 | 菜鸟教程 (runoob.com)

ArrayList 和 LinkedList 区别

List 是 Java 的一个接口，常见的实现类有**ArrayList**和**LinkedList**。

  • 底层：前者的底层数据结构是数组，后者是链表
  • 添加删除：ArrayList插入删除受元素位置影响，LinkedList是链表不受影响。
  • 查找：ArrayList支持高效随机访问，LinkedList不支持。

ArrayList 动态扩容*

ArrayList相比 List 的优点在于动态扩容。在源码中，有一个grow方法，每一次扩容原来的 1.5 倍，扩容完后调用arraycopy对数组进行拷贝。
ArrayList增删底层调用的copyOf()被优化过，现代 CPU 对内存可以块操作，增删一点不比LinkedList慢。

ArrayList 与 Vector 区别，前者为什么取代后者？

  1. 线程安全：Vector底层使用Synchronized实现线程同步，相比于ArrayList是线程安全的。
  2. 性能：但也因此Vector在性能上不如ArrayList。
  3. 扩容：Vector扩容为原来的 2 倍，而ArrayList只有 1.5 倍。

所以，在不需要线程安全的情况下优先使用ArrayList。

CopyOnWriteArrayList*

线程安全：CopyOnWriteArrayList。（在并发环境下需要使用线程安全的ArrayList进行修改操作，官方不推荐使用Vector）

HashMap，HashTable 区别

  • 线程安全：HashMap非线程安全，HashTable线程安全，内部方法经过synchronized，但！HashTable基本被淘汰，要保证线程安全请使用ConcurrentHashMap。
  • 效率：因为线程安全问题，HashMap效率比HashTable高
  • 底层：JDK1.8 后优化了HashMap解决哈希冲突，链表大于阈值 8 时（数组小于 64 先进行数组扩容），链表转为红黑树，增加搜索效率，Hashtable没有这个机制

HashSet 如何检查重复*

摘自《Head first java》第二版：

当你把对象加入 HashSet 时，HashSet 会先计算对象的 hashcode 值来判断对象加入的位置，同时也会与其他加入的对象的 hashcode 值作比较，如果没有相符的 hashcode，HashSet 会假设对象没有重复出现。但是如果发现有相同 hashcode 值的对象，这时会调用 equals()方法来检查 hashcode 相等的对象是否真的相同。如果两者相同，HashSet 就不会让加入操作成功。

重写 equals 方法为什么要重写 hashCode

**hashcode()**与**equals()**的相关规定：

  1. 如果两个对象相等，则hashcode 一定也是相同的
  2. 两个对象相等，对两个equals() 方法返回 true
  3. 两个对象有相同的hashcode值，它们也不一定是相等的

如果没有重写hashCode()，即使equals()判断为相同的两个对象，在加入HashSet时也不会被认为是重复对象。

Map 的几种实现类

Map 是 Java 的一个接口，常见的实现类HashMap、LinkedHashMap，TreeMap、ConcurrentHashMap。

HashMap 数组+链表/红黑树
LinkedHashMap 数组+链表+双向链表
TreeMap 红黑树
ConcurrentHashMap 数组+链表/红黑树

HashMap

**HashMap**有几个构造方法，最主要的是指定初始值以及负载因子的大小，默认大小为 16，负载因子为 0.75，大小只能是 2 次幂的。
负载因子决定哈希表的扩容和哈希冲突。负载因子太高意味着哈希冲突概率增高，更耗时。
数组大于 64 且链表大于 8 时，链表改为红黑树，当红黑树大小为 6 时退化为链表。链表查询时间复杂度 O(N)，插入 O(1)。红黑树查询插入 O(logN)。

HashMap 长度为什么必须是 2 的幂次方

首先查看HashMap源码中的putVal方法

注意这个(n - 1) & hash的作用：在HashMap中，为了存取高效，需要尽量减少碰撞，即将数据均匀分配在数组中。但是Hash值的范围有 40 亿映射空间（-21474836482147483647），无法直接存储，需要进行取模计算下标，因此有了(n - 1) & hash这个计算方法，因为采用了二进制操作符&，显而易见 HashMap 的长度必须是 2 的幂次方了

ConcurrentHashMap 和 HashTable 区别

上面说HashTable基本上被淘汰了，要保证线程安全请使用ConcurrentHashMap，显然在功能上已然取代了前者，两者的区别主要体现在实现线程安全的方式不同

  • 底层数据结构：
    • ConcurrentHashMap在 JDK1.7 中采用分段数组+链表的实现，JDK1.8 采用数组+链表/红黑二叉树
    • HashTable仅采用数组+链表的形式
  • 实现线程安全方式：
    • ConcurrentHashMap在 JDK1.8 使用Node数组+链表+红黑树的数据结构，并发控制使用 synchronized 和 CAS 来操作。
    • 使用synchronized保证线程安全，效率低下
