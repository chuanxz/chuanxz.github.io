如何写好简历

发表于 2022-05-11 更新于 2022-06-02 分类于 面试 阅读次数：

意识到如何写好简历这件事在面试中的重要性，毕竟简历写的不好，体现不出自己的能力的话，连面试机会都没有，其他更是空谈了。而认真对待一份简历，还可以表现出自己对这件事情，面试这家公司的态度，因此认真梳理了一些网上找到的写简历技巧，目前内容是针对Java 工程师。

关于简历

本质：证明自己的“技术能力&成长潜力”。

如何证明技术能力？

  1. 有大厂背景
  2. 热门开源软件发起人或贡献者
  3. 开源框架原理与源码烂熟于胸
  4. 主要解决过大数据高并发实际问题
  5. 能描述技术方案核心关键点
  6. 团队中主导解决过线上重大故障

如何证明成长潜力？

  1. 学历
  2. 学习态度，混迹开源社区
  3. 算法刷题
  4. 获奖证书

STAR 法则和 FAB 法则

在写简历时涉及的两个重要法则，可以作为标准参照。
STAR 法则
Situation-情景，Task-任务，Action-行动，Result-结果

  • S：事情在什么情况下发生？
  • T：你的任务是什么？
  • A：你做了什么？
  • R：最终结果怎么样？

FAB 法则
Feature-特征，Advantage-优势，Benefit-好处

  • F：你的特征是什么？
  • A：你比别人优势在哪里？
  • B：雇佣你会得到什么好处？

简历模板&招聘软件

简历模板
锤子简历：之前用过还不错
超级简历：有合伙企业，简历评分，简历推荐，相当于半个招聘软件了
木及简历：markdown 编辑，挺好看的，推荐

打造完简历—>招聘软件：智联，BOSS，拉钩

具体内容

个人信息

  • 基本信息：姓名、年龄、电话、籍贯、联系方式、邮箱
  • 潜在加分项：Github、博客地址（如果有内容…）

专业技能

问问自己会什么，意向公司需要什么。
没必要写一些杂七杂八和应聘岗位无关的技能，有时候全才也是庸才。

  • 熟练程度：精通别乱写

工作履历

建议使用倒叙

  • 在职时间：如实，编造被拉黑得不偿失
  • 所属行业：
  • 公司名称：
  • 担任职位：

实习经历/项目经历

这块算是一个重点，突出做过的项目亮点

  • 项目名称：
  • 项目描述：简短介绍，让面试官和 HR 知道大概是个什么类型的项目
  • 涉及技术：枚举亮点技术，尽量能匹配上招聘要求
  • 个人职责：
  • 难点收获：

⭐ 常见项目问题

  • 负责项目哪块内容？
  • 项目难点痛点？如何解决？
  • 使用 XX 技术栈时有没有什么坑？怎么解决的？
  • 印象比较深的 Bug？
  • 遇到过 XX 情况么？怎么解决优化的？
  • 根据哪些指标针对性优化？

奖项证书

校招可以多写点，证明自己的学习态度和能力
社招选填部分，有含金量的

个人评价

我本来以为不需要这个自夸的环节…后来发现还是挺有必要的，
个人评价属于自我解读，需要简洁，突出自身亮点

注意事项

  1. 可以先通过一些小公司练手。
  2. 简历使用 PDF 投递。
  3. 少讲废话，突出亮点。
  4. 专业技能和做过的项目有重合度。

技巧

  1. 面试和工作是两回事，聪明的人把面试官往自己擅长的领域领，其他人则被面试官牵着鼻子走。
  2. 投递简历之前，根据公司招聘信息微调简历的描述。
