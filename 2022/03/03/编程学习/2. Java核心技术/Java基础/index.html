String 为什么不可变？有什么好处？

为什么：

  1. String 里的 value 使用 final 不可再次赋值，本身 final 不可继承
  2. 没有暴露成员变量，都是 private
  3. 内部方法本身不会修改 value

好处：

  1. 安全+线程安全
  2. 节省空间
  3. 性能，只做一次 hashCode

重写 equals 方法为什么一定要重写 hashcode？

视频：https://www.bilibili.com/video/BV1jb4y1r7yd?spm_id_from=333.337.search-card.all.click
背景：equals 和 hashcode 是 object 类的两个基础方法
该问题可分为两个小问题：
Q1：为什么要重写 equals？

public boolean equals(object obj){
return (this == obj);
}

equals 的源码是在判断两个对象是否具有相同的引用，在多数情况下这种判断是没有意义的，返回值为 false。
Q2：重写 equals 时为什么要重写 hashcode？
因为 hashcode 用于获取地址，需要与 equals 逻辑一致，如果不重写的话，无法获取到值。
假设通过姓名和年龄判断是否为同一个人（equals），
那么具有相同姓名和年龄，其 hashcode 值也必然一致。

NIO

no-blocking io，用于提高速度
区别
传统 IO 一次处理一个字节，是阻塞的
NIO 以块（缓冲区）处理数据，可非阻塞
NIO 三个核心部分

  • Buffer：缓冲区用于存储数据
  • Channel：管道用于运输数据
  • Selector：选择器检查多个 Channel 的状态变更

泛型通配符

参考：https://www.cnblogs.com/minikobe/p/11547220.html

常用 T，E，K，V，？

  • T 表示具体 Java 类型
  • E 表示 Element
  • KV 表示键值对
  • ？表示不确定 Java 类型

？ 无界通配符

上界通配符<? extends E>

表示参数化的类型可能是所指定的类型，或者是此类型的子类。

下界通配符<? super E>

表示参数化的类型可能是所指定的类型，或者是此类型的父类型，直至 Object

? 和 T 的区别

T 是一个 确定的 类型，通常用于泛型类和泛型方法的定义，？是一个 不确定 的类型，通常用于泛型方法的调用代码和形参，不能用于定义类和泛型方法。
